WeatherStar 4000

Host WeatherStar4000 locally. Do not point production channels at a public WeatherStar URL. That puts unnecessary load on someone else's server and can stop working without notice.

Run your own WeatherStar4000 instance on your LAN or the same machine as Jellyfin, then enter that base URL below. FinTV appends each weather channel's latitude and longitude as ?lat=…&lon=….

For testing only, you may use a public hosted demo (not for production):

  • https://weather.jmthornton.net/
  • https://4000.weatherstar.dev/
  • https://weatherstar3000.netbymatt.com/

Self-host a WeatherStar build from one of these projects, then paste your local base URL above:

  • TheGoldDiamond9 / weatherstar4000
  • thornjad / weatherstar4k
  • netbymatt / ws3kp (WeatherStar 3000)

Example local URL: http://127.0.0.1:8080 or http://192.168.1.50:8080. Do not include lat or lon here; FinTV adds them from each weather channel.

Jellyfin Live TV URLs

Jellyfin fetches these URLs from the server itself. They must not require login and must use an address Jellyfin can reach (not localhost unless Jellyfin runs on the same machine and resolves it).

When a viewer tunes in and the channel has no media scheduled, FinTV shows an EBS slate and plays random audio from the selected source. Create a library named Background Music or choose any music library.
